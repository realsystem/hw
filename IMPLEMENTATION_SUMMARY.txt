WHAT WAS CREATED
================================================================================

1. INVENTORY SYSTEM
   File: poc/inventory.yml
   - Single source of truth for all servers
   - Defines: hostnames, IPs, MAC addresses, IPMI credentials, hardware specs
   - Role-based (OSD vs MON nodes)
   - Example: 6 nodes (3 OSD, 3 MON)

2. TERRAFORM INFRASTRUCTURE (with terraform-coop/foreman provider)
   Location: poc/terraform/

   00-provider/
   - Foreman provider setup and connection testing
   - Variables: foreman_hostname, foreman_protocol, credentials

   01-foreman-config/
   - Creates hostgroups (Ceph OSD, Ceph MON)
   - Creates subnets (Provisioning, Management)
   - Creates partition tables (Debian preseed)

   02-node-provision/
   - Loads inventory.yml
   - Powers on servers via IPMI
   - Provisions hosts in Foreman
   - Generates Ansible inventory

3. ANSIBLE CONFIGURATION
   Location: poc/ansible/

   Playbooks:
   - ceph_baseline.yml - Main configuration playbook
   - validate.yml - Validation tests (future)

   Roles (6 total):
   - common: Package installation, timezone, services
   - network: Network tuning, sysctl, MTU configuration
   - kernel: Kernel parameters, THP disable, CPU governor
   - storage: I/O schedulers, disk configuration
   - time: Chrony NTP setup
   - monitoring: Prometheus node_exporter

   Config:
   - ansible.cfg: Sets roles_path, disables host_key_checking

4. TEST INFRASTRUCTURE (Docker-based)
   Location: poc/test-infra/

   Option 1: Ansible Testing
   - PXE server (dnsmasq) for DHCP/DNS/TFTP
   - 3 simulated nodes (SSH-accessible containers)
   - Hardware facts simulation
   - File: docker-compose.simple.yml

   Option 2: Terraform Testing
   - Mock Foreman API (Flask-based) at http://localhost:3000
   - 3 simulated nodes (same as Option 1)
   - Supports: architectures, OS, hostgroups, subnets, ptables, hosts
   - File: docker-compose.terraform.yml

   Scripts:
   - setup.sh: Interactive setup (choose mode 1 or 2)
   - inventory-sim.yml: Ansible inventory for test nodes

   Mock API:
   - foreman-mock/app.py: Flask application
   - Supports v1 (/api/) and v2 (/api/v2/) endpoints
   - HTTP Basic Auth: admin/changeme123
   - In-memory storage (resets on restart)

5. AUTOMATION SCRIPTS
   Location: poc/scripts/

DEPLOYMENT WORKFLOW
================================================================================

FOR TESTING (No Real Hardware Required):

Step 1: Test Ansible Playbooks
  cd poc/test-infra
  ./setup.sh                     # Choose option 1 (Ansible)
  cd ../ansible
  ansible-playbook -i ../test-infra/inventory-sim.yml playbooks/ceph_baseline.yml

Step 2: Test Terraform Configurations
  cd ../test-infra
  docker-compose -f docker-compose.simple.yml down
  ./setup.sh                     # Choose option 2 (Terraform)
  cd ../terraform/00-provider
  terraform init
  terraform plan
  terraform apply
  cd ../01-foreman-config
  terraform init && terraform apply

FOR PRODUCTION DEPLOYMENT:

Step 1: Install Foreman on Bootstrap Host
  # See poc/FOREMAN_SETUP_GUIDE.md
  sudo foreman-installer --enable-foreman-plugin-ansible

Step 2: Configure Inventory and Credentials
  cd poc
  vim inventory.yml              # Add your servers
  cd terraform/00-provider
  cp terraform.tfvars.example terraform.tfvars
  vim terraform.tfvars           # Add Foreman URL, credentials

Step 3: Configure Foreman with Terraform
  terraform init
  terraform apply
  cd ../01-foreman-config
  terraform init && terraform apply

Step 4: Provision Nodes
  cd ../02-node-provision
  terraform init
  terraform plan                 # Review nodes to provision
  terraform apply                # Powers on via IPMI, provisions OS

Step 5: Configure Nodes with Ansible
  cd ../../ansible
  ansible-playbook -i inventory/terraform_hosts.yml playbooks/ceph_baseline.yml

Step 6: Validate
  ansible -i inventory/terraform_hosts.yml all -m ping
  # Check node status, ready for Ceph deployment

TERRAFORM PROVIDER DETAILS
================================================================================

Provider: terraform-coop/foreman
Version: >= 0.1
Registry: https://registry.terraform.io/providers/terraform-coop/foreman

Configuration format:
  provider "foreman" {
    server_hostname      = "localhost:3000"  # or "foreman.example.com"
    server_protocol      = "http"            # or "https"
    client_username      = "admin"
    client_password      = "changeme123"
    client_tls_insecure  = "true"            # string, not boolean
    provider_loglevel    = "INFO"
    provider_logfile     = "terraform-provider-foreman.log"
  }

Resources available:
  - foreman_hostgroup
  - foreman_subnet
  - foreman_partitiontable (note: no underscore)
  - foreman_host
  - foreman_architecture (data source)
  - foreman_operatingsystem (data source)
